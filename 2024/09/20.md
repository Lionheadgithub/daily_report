## わかったこと
•  プッシュした後やコンフリクトしそうな時と作業の履歴を残したいならマージ(merge)を使う

• プッシュしていないローカルの変更と履歴をきれいにしたいならリベース(rebase)を使う

• git commit --amendで直前のコミットをやり直す(リモートリポジトリにpushしたコミットはやり直してはいけない)

• git rebase -i<コミットID>で複数のコミットをやり直す。HEAD~で1番目の親を指定する。HEAD^でマージした場合の2番目の親を指定する。

• git tag  コミットを参照しやすくするためにわかりやすい名前をつけるのがタグ。よくリリースポイントに使う。

• タグには2種類ある

注釈付き(annotated)版：名前をつけられる、コメントをつけられる、署名をつけられる

軽量(lightweight)版：名前をつけられるのみ

• git show[タグ名]でタグのデータを表示する

• タグをリモートへ送信するにはgit pushコマンドで別途してする git push[リモート名][タグ名]

• 作業が途中でコミットはしたくないけれど別のブランチで作業しないといけない場合は作業を一時避難する。　git stash

• git stash listで避難した作業の一覧を表示する

• git stash applyで最新の作業を復元する

• git stash dropで最新の作業を削除する
