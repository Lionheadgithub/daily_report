## わかったこと
• 実際のGitでのバージョン管理の流れとして作
成した変更分をリポジトリに保存する際には、
git add .というコマンドを叩いて、stagingエリ
アに変更分を追加する。

• stagingエリアはリポジトリに変更分を保存す
る前に、今回の変更分が正しいかチェックする
場所。stagingエリアにて変更分が正しい場合
はgit commitコマンドにて変更分をリポジトリ
に保存します。

• 保存の形式はスナップショットにて保存され
る。スナップショットとは、ファイルの状態を
丸ごと保存することを指す。

• git commitした際には、ID、メッセージ（編集した意図）、日時、編集者、スナップショットが
保存されます。

• echo sample1 > a.txt、このコマンドはファイ
ルを作成しつつ書き込む処理になる。

• git statusコマンドを叩いて差分が赤く表示さ
れる、この変更文を保存するためにstagingエリ
アに追加する必要がある。

• stagingエリアに追加するためにはgit addコマ
ンドを使用する。

例: a.txtのみの場合はgit add a.txt

• 複数指定の場合はgit add a.txt b.txtとスペース
を開けて指定する

• 差分すべてを追加したい場合はgit add .とする
とすべての差分を追加することができる。こ
ちらは実務でよく使用することになる。
